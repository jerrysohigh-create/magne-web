L1

Trusted Execution Isolation

The first layer of protection operates at the system execution level. Sensitive operations are isolated in a trusted execution environment, separate from the main operating system. This means even if the main OS is compromised, the isolated execution environment remains protected.

This is not software-based isolation — it is architectural separation built into the hardware foundation of the device. It defines which processes can access what, and creates hard boundaries around the most sensitive operations.

L2

Secure Element Protection

The second layer is where private keys live. Unlike software wallets where keys are stored in application memory, MAGNE's secure element is a dedicated hardware component designed specifically for cryptographic key storage and signing operations.

Private keys never leave the secure element. When a transaction needs to be signed, the signing operation happens inside the secure boundary — the main processor never has access to the raw private key. This is the same architecture used in hardware security modules used by financial institutions.

L3

Encrypted NFC Recovery

The third layer addresses the oldest problem in digital asset management: what happens when you lose access to your device? NFC-based encrypted backup extends protection beyond the hardware itself.

When you set up MAGNE, an encrypted backup is generated and stored on an NFC tag or another device. This backup contains the information needed to recover your assets — without exposing your private keys in plain text. Recovery is physical, offline, and does not require cloud access or a third party.

How MAGNE Compares to Software Wallets

Software wallets make trade-offs between usability and security. MAGNE's hardware-backed architecture addresses each one.

Aspect Software Wallets MAGNE
Key Isolation Keys stored in app memory, accessible to OS Keys isolated in dedicated secure hardware
Signing Environment Software-based signing on main processor Hardware signing inside secure element boundary
Backup Method Seed phrase (can be stolen, lost, guessed) NFC encrypted backup — physical, offline, no seed exposure
Device Exposure Entire device must be trusted and secure Only secure element needs protection — main OS can be compromised
Recovery Model Depends on seed phrase + third-party tools Offline NFC recovery without cloud or third-party
Certification No hardware security certification CC EAL2+ / EAL6+ certified secure element

Security Chip Components

At the heart of MAGNE's security solution are purpose-built components designed for hardware-grade protection.

COMPONENT

UTEE — UNISOC TEE

Custom Trusted Execution Environment developed in partnership with UNISOC. UTEE isolates sensitive data and operations from the main system, ensuring cryptographic keys and user data are securely processed within a trusted environment. CC EAL2+ certified.

COMPONENT

HYPERSEED N60 (CIU98M50)

Custom security solution combining the CIU98M50 processor with an integrated NFC module. Offers secure cryptographic operations, key isolation, and offline key recovery. CC EAL6+ certified.

COMPONENT

HYPERSEED 72B (CIU9872B)

Software-enhanced version of the CIU9872B, providing additional encryption and key recovery functionality. CC EAL6+ certified.

Certified Security Standards

MAGNE's security components meet the highest international security certification standards. UTEE is certified CC EAL2+; HYPERSEED N60 and 72B are certified CC EAL6+, providing robust protection against both physical and software-based attacks.
