### Critical Technical Discovery
- The EN master file (`/magne-web/magne-home.html`) is a **double-DOCTYPE embedded document**:
  - Outer shell (lines 0-34175): GitHub Pages redirect/meta frame + outer HTML shell
  - Inner embedded doc (from 2nd DOCTYPE onward): The actual page content
  - `</head>` appears at position ~34176, `<body` at ~34176, then DOCTYPE + embedded `<head>...</head><body>...</body>`
- The embedded body contains the **full page content including nav and footer**
- Original approach of translating `embedded_body` text nodes was correct, BUT the extracted body still had English nav links because nav link text lives inside `<a>` tags as text children
- Nav items (Home/Phone/Security/etc.) were NOT being translated because the translation function only processed text between tags, not text inside tag children when the whole tag was one chunk
- **Fix**: Post-processing string replacements for nav items (`>Home<` вҶ’ `>йҰ–й Ғ<`, etc.)
